# Sprint 9 - Full QA Testing Suite (Web, Mobile, API Testing)

## 📌 Project Overview
This final project focused on end-to-end QA testing across multiple platforms, including web, mobile, and API layers. The goal was to simulate a real-world QA engineer workflow by designing and executing complete test coverage for a scooter rental application.

This sprint combined all previously learned skills including test design techniques, mobile testing, API validation, and defect reporting into a unified QA deliverable.

---

## 🧪 What I Did
- Created comprehensive test cases for a scooter rental web application using requirements analysis
- Applied Equivalence Partitioning (EP) and Boundary Value Analysis (BVA) to design test scenarios
- Executed manual web testing and documented results in structured Excel sheets
- Designed mobile test cases and executed them using Android Studio (Pixel 5 emulator)
- Tested push notifications and internet connectivity scenarios on mobile
- Validated API endpoints using Postman:
  - POST /api/v1/courier (courier creation)
  - DELETE /api/v1/courier/:id (courier deletion and data cleanup validation)
- Verified API response codes, payload structure, and business logic behavior
- Organized all test results into structured documentation for reporting
- Logged defects in Jira for identified issues across web, mobile, and API layers
