# Sprint 8 - Page Object Model (POM) Framework with Selenium WebDriver

## 📌 Project Overview
This project focused on building a scalable UI test automation framework using the Page Object Model (POM) design pattern with Selenium WebDriver in Python.

The goal was to separate test logic from page interactions to improve maintainability, reusability, and readability of automated tests for the Urban Routes application.

---

## 🧪 What I Did
- Designed and implemented a Page Object Model (POM) structure for UI automation
- Created a dedicated page class (UrbanRoutesPage) to store locators and UI interaction methods
- Refactored test cases to use page methods instead of direct Selenium calls
- Organized locators using By.ID, By.XPATH, and other Selenium strategies
- Built reusable methods for core user actions (e.g., setting route, selecting plan, entering phone number, payment flow, and ordering taxi)
- Implemented explicit waits using WebDriverWait and Expected Conditions for stable test execution
- Separated test data, page logic, and test cases for clean architecture
- Executed end-to-end automated test scenarios using Pytest
