# Sprint 5 - SQL Testing & Database Validation

## 📌 Project Overview
This project focused on validating data integrity in a venture capital database using SQL queries.

The goal was to ensure that backend data matched expected business rules by writing and executing SQL queries directly in a controlled testing environment.

---

## 🧪 What I Did
- Executed SQL queries to validate database accuracy after application updates
- Queried multiple tables including companies, funding rounds, acquisitions, and investors
- Applied filtering, aggregation, and grouping techniques to analyze data
- Verified business rules such as:
  - Closed startups appearing correctly in the database
  - Correct calculation of total funding amounts
  - Accurate filtering of acquisitions by date and payment type
  - Identification of influencers using pattern matching (LIKE operator)
- Performed data validation for geographic and financial reporting logic
- Ensured query outputs matched expected test scenarios

---

## 🛠️ Tools Used
- SQL (SELECT, WHERE, GROUP BY, HAVING, ORDER BY)
- Aggregate functions (SUM, COUNT, AVG)
- Pattern matching (LIKE operator)
- TripleTen SQL web-based environment
- Database testing methodology

---

## 🧠 Skills Demonstrated
- SQL query writing and execution
- Data validation and integrity testing
- Business logic verification through database queries
- Aggregation and grouping analysis
- Filtering and pattern matching
- Understanding relational database structure
- QA backend testing fundamentals

## 🎯 Outcome
This sprint strengthened my ability to validate backend systems using SQL. I learned how to extract meaningful insights from structured data and verify that application logic is correctly reflected in the database.
