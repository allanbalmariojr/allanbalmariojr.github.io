# Sprint 7 - Python Automation Setup (Selenium WebDriver Foundation)

## 📌 Project Overview
This project focused on setting up the foundation for UI test automation using Python and Selenium WebDriver. The goal was to prepare a scalable automation structure that would later support full end-to-end test execution in a Page Object Model (POM) framework.

This sprint introduced core automation setup concepts, including project structure, helper utilities, and initial test scaffolding for the Urban Routes application.

---

## 🧪 What I Did
- Created a Python-based automation project structure for UI testing
- Set up Selenium WebDriver environment for browser automation
- Added helper functions to support test execution (e.g., retrieving phone confirmation codes and checking server availability)
- Configured test data storage using a separate constants/data file
- Built initial test scaffolding using Pytest structure
- Prepared reusable test functions for future automation implementation
- Validated that the application server was reachable before test execution
- Committed and maintained version control using GitHub

---

## 🛠️ Tools Used
- Python
- Selenium WebDriver
- Pytest framework (test structure setup)
- GitHub (version control)
- Chrome WebDriver
- IDE (development environment setup)

## 🧠 Skills Demonstrated
- Test automation setup and project structuring
- Selenium WebDriver configuration
- Separation of test data from test logic
- Writing reusable helper utilities
- Understanding of test framework architecture
- Version control using Git/GitHub
- Preparation for Page Object Model (POM) design

---

## 🎯 Outcome
This sprint established the foundation for scalable UI test automation. I learned how to structure an automation project properly, separate concerns between test data and test logic, and prepare reusable components that support maintainable and scalable QA automation frameworks.
