# Sprint 2 - Test Planning & Test Case Design (Equivalence Partitioning & Boundary Value Analysis)

## 📌 Project Overview
This project focused on analyzing software requirements for a fake ride-sharing application and translating them into structured test cases using formal QA test design techniques.

The goal was to ensure complete test coverage of input fields and system behavior by applying Equivalence Partitioning (EP) and Boundary Value Analysis (BVA) to identify valid and invalid test scenarios.

---

## 🧪 What I Did
- Analyzed ride-sharing application requirements and broke them down into atomic, testable components
- Identified valid and invalid input classes using Equivalence Partitioning (EP)
- Defined boundary values for input fields using Boundary Value Analysis (BVA)
- Designed both positive and negative test cases based on identified conditions
- Created structured test case documentation in Excel format for execution readiness
- Prepared test cases for later execution in Sprint 3 and beyond
