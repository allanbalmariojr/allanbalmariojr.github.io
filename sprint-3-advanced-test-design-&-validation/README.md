# Sprint 3 - Advanced Test Design & Validation (DevTools, JSON, Network Inspection)

## 📌 Project Overview
This project focused on testing a ride-sharing application by validating frontend behavior and backend responses using browser developer tools.

The goal was to strengthen test coverage by combining Equivalence Partitioning (EP) and Boundary Value Analysis (BVA) with real-time debugging techniques using Chrome DevTools, including network inspection and JSON response validation.

---

## 🧪 What I Did
- Analyzed application requirements and extended test cases from Sprint 2 into more advanced scenarios
- Applied Equivalence Partitioning (EP) and Boundary Value Analysis (BVA) to refine test coverage
- Performed functional testing of UI behavior in a ride-sharing application
- Used Chrome DevTools to inspect network requests and responses
- Validated JSON payloads to ensure correct data exchange between frontend and backend
- Identified and documented defects based on UI and API inconsistencies
- Logged bugs and issues in structured format for tracking and resolution

---

## 🛠️ Tools Used
- Google Chrome DevTools (Network tab, Console, JSON inspection)
- Microsoft Excel (Test case documentation)
- Manual testing techniques
- QA test design techniques (EP, BVA)
- Basic API/response validation concepts

---

## 🧠 Skills Demonstrated
- Advanced test case refinement and execution
- Frontend and backend validation through DevTools
- Network request and response analysis
- JSON data inspection and validation
- Debugging and defect identification
- Functional and integration testing concepts
- QA documentation and structured reporting

---

## 🎯 Outcome
This sprint improved my ability to validate not only UI behavior but also underlying network and data layer responses. It strengthened my understanding of how frontend actions map to backend API calls, which is essential for effective modern QA testing in web applications.
