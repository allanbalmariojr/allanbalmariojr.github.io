# Sprint 4 - API Testing (Postman, Swagger, and Backend Validation)

## 📌 Project Overview
This project focused on API testing for a ride-sharing application by validating backend endpoints responsible for handling core application functionality.

The goal was to ensure that API requests and responses behaved correctly by testing endpoints using Postman, reviewing API documentation, and verifying data integrity through structured test cases.

---

## 🧪 What I Did
- Analyzed API requirements using Swagger documentation and apiDoc references
- Designed test cases for backend endpoints based on expected request and response behavior
- Performed API testing using Postman
- Validated response status codes, response bodies, and data correctness
- Tested functional scenarios including valid and invalid request inputs
- Verified backend behavior aligned with frontend expectations
- Documented test cases and results in Excel format
- Logged defects when API responses did not match expected behavior

---

## 🛠️ Tools Used
- Postman (API testing and request validation)
- Swagger UI / apiDoc (API documentation reference)
- Microsoft Excel (Test case documentation)
- Manual API testing techniques
- QA test design techniques (EP, BVA applied conceptually)

---

## 🧠 Skills Demonstrated
- API testing fundamentals
- Understanding RESTful services (GET, POST, etc.)
- Request/response validation
- Status code validation (200, 201, 400, etc.)
- JSON response inspection
- Test case design for backend systems
- Bug reporting and QA documentation

---

## 🎯 Outcome
This sprint strengthened my ability to test backend systems independently of the UI. I gained hands-on experience validating APIs using Postman and learned how to ensure data consistency between backend services and frontend application behavior.
