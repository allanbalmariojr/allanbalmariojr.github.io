# Sprint 6 - Mobile Testing (Urban Lunch App)

## 📌 Project Overview
This project focused on mobile application testing for the Urban Lunch app using an Android emulator. The goal was to ensure the application met functional requirements by validating UI behavior, user flows, and edge cases based on provided specifications and design layouts.

I created structured test coverage using checklists, executed manual tests on a mobile environment, and reported defects through a QA workflow similar to real-world mobile testing processes.

---

## 🧪 What I Did
- Analyzed Urban Lunch app requirements and Figma layouts to understand expected behavior
- Created a structured mobile testing checklist covering key functional areas
- Installed and ran the application on an Android emulator (Pixel 5)
- Performed manual functional testing across multiple user flows
- Verified UI behavior against requirements and design specifications
- Logged defects in Jira when application behavior did not match expected results
- Marked test cases as PASSED or FAILED based on observed outcomes
- (Optional) Prepared a test summary report evaluating overall app stability

---

## 🛠️ Tools Used
- Android Studio Emulator (Pixel 5 device simulation)
- Jira (defect reporting)
- Microsoft Excel (test checklist documentation)
- Manual mobile testing methodology
- UI/UX requirement validation (Figma reference)

- Mobile application testing (Android)
